## ✨ Highlights

- **Website-style dashboard** built with React + Vite
- **Rule-based prediction API** powered by FastAPI
- **Actionable outputs**: risk level, confidence, timing, advice, and SMS text
- **Low-connectivity design** for field-friendly communication

## 🧠 How it works

The frontend sends weather inputs to the backend `/predict` endpoint.
The backend applies lightweight rules:

- High rainfall + high soil moisture → **Flood risk**
- Very low rainfall + high temperature → **Drought risk**
- High wind or heavy rain → **Storm warning**

Then it returns:

- risk level
- confidence score
- timing window
- farmer advice
- SMS-ready message

## 🗂️ Project structure

- `/backend/main.py` — prediction API
- `/backend/requirements.txt` — backend dependencies
- `/frontend/src/App.jsx` — dashboard UI
- `/frontend/package.json` — frontend dependencies

## 🚀 Run locally

### 1) Start backend

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload
```

### 2) Start frontend

Open a second terminal:

```bash
cd frontend
npm install
npm run dev
```

Then open the local Vite URL in your browser.

## 🎬 Demo flow

1. Open the dashboard and review sample alerts.
2. Explain alerts are generated by the prediction API.
3. Change rainfall / temperature / soil moisture values.
4. Click **Generate Prediction**.
5. Show updated risk card and SMS preview.
6. Close with the farmer impact narrative for low-connectivity regions.
